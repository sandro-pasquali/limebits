Javascript: A History

Javascript is the most popular programming language in use today. This popularity has nothing to do with the superiority of Javascript as a programming language. Javascript is the scripting language natively supported by all browsers, and as such is the language used to create the more advanced functionality and visual effects which anyone who visits a modern website enjoys. As the number of websites grows the number of applications written using Javascript grows, and this means more jobs for those who know Javascript. If you are building websites, you must be aware of how Javascript works.

The Start of a Revolution

In early 1995 Brendan Eich was hired by the Netscape Corportation to design a simple scripting language which would assist non-programmers in using Netscape Navigator's newly added support for Java applets. As one person put it:

Brendan Eich convinced his pointy-haired boss at Netscape that the Navigator browser should have its own scripting language, and that only a new language would do, a new language designed and implemented in big hurry, and that no existing language should be considered for that role.
— Douglas Crockford, Senior Javascript architect, Yahoo

The language he created was initally named "Livescript". This was later changed to "Javascript", a marketing mistake that saddled the language with a fully mistaken association to Java, and with comparisons to that language and other "serious" languages along its functionality and design. All of this confusion kept Javascript in the "toy language" category for quite some time, even as its use steadily grew, at first in providing simple dynamic effects for webpages (image rollovers on mouse events), and later as the gateway to creating full-blown applications in a browser.

Enter Redmond

Somewhere around this time Microsoft had decided to enter the browser game, and it too created a scripting language for their upcoming browser, IE3. Microsoft created the VBScript language (which hooked into embeddable components on Windows platforms) and a port of Javascript, "JScript". They then released their IE3 browser, which notably was the first browser to offer support for CSS (Cascading Style Sheets). The incompatibility between IE3 and Netscape Navigator in their implementation of Javascript was aggravated with the release 4.0 browser generation, in which both manufacturers created completely proprietary document object models (DOM). Thus began the storied "browser wars".

One Step Forward, Two Steps Back

As developers began to find, fix, and work around the bugs, security flaws, and incompatabilities between the browsers, the Internet took off. While various individuals, companies, and consortiums were working to standardize Javascript and the DOM, very real people were already spending very real money fueling Web 1.0. Developers struggled to create libraries which tried to smoothed the differences between browsers. The difficulty of doing so, aggravated by the constantly changing implementations the DOM and Javascript itself, resulted in either single-browser specific websites, or in a general avoidance of any fancy scripting in the websites which were gaining huge numbers of new users and traffic. New technologies came into the picture aimed at providing a standard and easy way to create dynamic websites, and the one technology that rose to prominence in this time was Macromedia Flash. Dynamic HTML websites, created by using Javascript to manipulate the still-flaky DOM, continued to be produced and their potential was clear, but the difficulty of implementing stable solutions which always worked across all browsers held back real developments in the field.

AJAX

Between 2000 and 2005 some successes were seen for companies which managed to build rather dynamic and powerful applications which worked in browsers. However, it was with the coining of the term AJAX by Jesse James Garrett that the current and lively interest in browser-based application development flourished (Ajax: A New Approach to Web Applications). Sometimes all that you need is a name.
