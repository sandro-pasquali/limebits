AXIS - Core Extensions

When the AXIS is loaded is automatically loads certain extensions. These extensions provide key functionality, such as Login control, Cookie management, and XHR requests. This is a (very)brief overview of these core extensions, with very basic examples of use. Individual extensions are discussed in their own sections. All core extensions are accessible via AXIS.[], such as AXIS.Cookies.

Cookies

Standard methods for managing cookies. 

AXIS.Cookies.create("myCookie","foo");
AXIS.Cookies.erase("myCookie");

User

Methods for getting current user information. This mostly has to do with user's authenticated state. 

// Check(bool) if user is logged in.
AXIS.User.isLoggedIn();

Loader

The Loader object manages XHR calls. It does useful things like ensuring the integrity of requests, notifying the viewer of load states, and so forth. It does not do the actual loading, which is left to the AXIS.XHR object. 

// GET a file
AXIS.Loader.load({
  method: 'GET',
  url:    'http://www.google.com'
});

XMLHTTP

The XMLHTTP object is the "engine" which handles XHR calls. It should not be used directly; you should always use the call API provided by either the AXIS.Loader or AXIS.WebDAV. It is useful to investigate the code, however, to see how these sorts of requests happen and are handled.

WebDAV

The WebDAV object provides an API allowing the developer to make WebDAV calls. 

AXIS.WebDAV.MKCOL({
  url:          'newfolder/',
  callback:     function(r) {
    alert('folder created');
  }
});

Login

This object manages the users authentication state. Login is automatic, so this object is only really used internally by the AXIS.
