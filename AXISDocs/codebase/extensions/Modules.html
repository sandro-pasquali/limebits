AXIS - Modules

Methods for loading modules into the current workspace. In order to use modules, you will need to add the Modules extension: 

  <script type="text/javascript" src="/!lime/root/library/limebits/latest/axis/AXIS.js" 
  extensions="Modules"></script>

The Module Object

Modules are accessed via a common definition.

ModuleObject
  • String provider : The module provider, serving as a grouping category.
  • String module : The name of the requested module.
  • Number version : The module version to load.
  • Callback onload : Function called module loads successfully. Passed whatever is returned by module initialization function.
  • Object options : Any module-specific options.

Creating Modules

Methods

load
  • ModuleObject
Returns Boolean indicating success or failure.

Loads a module. This documentation is itself a module, the `Docs` module, ie: 

      AXIS.Modules.load({
       	provider: 'local',
       	module:   'Docs',
       	options:  {
       	            rootPath: '../../',
       	            },
         	          groupURI: 'http://groups.google.com/group/limebits/post'
       	          }
      });
